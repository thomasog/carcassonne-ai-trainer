Como publicar no GitHub e rodar o treinamento automático GRÁTIS

Visão geral

O GitHub Actions oferece 2.000 minutos/mês gratuitos para repositórios públicos (e privados na conta gratuita). O workflow roda a cada 6 horas e usa no máximo 120 minutos por execução — então, em tese, você treina continuamente sem custo.

RecursoLimite gratuitoUso estimado
Minutos de Actions (Ubuntu)2.000 min/mês~120 min × 4/dia = 480 min/dia
Armazenamento de Artifacts500 MB~1–5 MB por run

Se ultrapassar 2.000 min/mês no plano gratuito, o workflow simplesmente para até o próximo mês. Nada é cobrado automaticamente.

Passo a passo

1

Crie uma conta no GitHub (se não tiver)

Acesse github.com e crie uma conta gratuita. Não precisa de cartão de crédito.

2

Crie um novo repositório

  • Clique no botão + no canto superior direito → New repository.
  • Nome sugerido: carcassonne-ai-trainer
  • Visibilidade: Public (para ter Actions ilimitado) ou Private (2.000 min/mês).
  • Marque Add a README file para que o repositório já seja inicializado.
  • Clique em Create repository.
3

Faça upload dos arquivos

Existem duas formas:

Opção A — Interface web (mais simples, sem instalar nada):

  • Na página do repositório, clique em Add file → Upload files.
  • Arraste a pasta AI Trainer/ inteira ou os arquivos individualmente.
  • Atenção: a estrutura de pastas precisa ser preservada. Para subir .github/workflows/train.yml pela interface web, você precisa criar o caminho manualmente: clique em Create new file, escreva .github/workflows/train.yml no campo de nome e cole o conteúdo do arquivo.
  • Commit direto na branch main.

Opção B — Git pela linha de comando (recomendado):

# Clone o repositório vazio
git clone https://github.com/SEU_USUARIO/carcassonne-ai-trainer.git
cd carcassonne-ai-trainer

# Copie os arquivos da pasta "AI Trainer/" para dentro do clone
# (ajuste o caminho conforme seu sistema)
cp -r "/caminho/para/AI Trainer/src" .
cp -r "/caminho/para/AI Trainer/.github" .
cp "/caminho/para/AI Trainer/package.json" .

# Commit e push
git add .
git commit -m "feat: adicionar AI Trainer inicial"
git push origin main
4

Ative o GitHub Actions

  • No repositório, clique na aba Actions.
  • Se aparecer uma mensagem pedindo para habilitar workflows, clique em I understand my workflows, go ahead and enable them.
  • Você verá o workflow Train Carcassonne AI listado.
5

Rode o treinamento manualmente pela primeira vez

  • Na aba Actions, clique em Train Carcassonne AI.
  • Clique em Run workflow (lado direito).
  • Você pode ajustar o tempo de orçamento e o tamanho da população, ou deixar os valores padrão.
  • Clique em Run workflow para iniciar.
O primeiro run executa o npm run smoke automaticamente. Se algum teste falhar, o workflow para e você verá o erro em vermelho.
6

Acompanhe o progresso

  • Clique no run em execução para ver os logs em tempo real.
  • O step Train imprime o fitness de cada geração no log.
  • Ao terminar, o step Upload results salva os arquivos como Artifact (botão Artifacts no canto inferior da página do run).
  • O step Commit results faz um commit automático na branch main com os arquivos atualizados em results/.
7

Verifique os resultados

  • No repositório, navegue até a pasta results/.
  • best-weights.json — os melhores pesos encontrados até agora.
  • leaderboard.json — top candidatos da última geração.
  • history.jsonl — histórico de todas as gerações.
  • summary.csv — tabela com fitness, winRate, avgMargin por run.
  • latest-run.json — resumo da última execução.
8

Treinamento agendado automático

O workflow já está configurado para rodar automaticamente a cada 6 horas via cron (0 */6 * * *). Não precisa fazer mais nada — depois do primeiro run manual, o treinamento continuará sozinho, acumulando os melhores pesos a cada ciclo.

O GitHub pode pausar workflows agendados em repositórios sem atividade por 60 dias. Para evitar isso, você pode fazer um pequeno commit manual de tempos em tempos, ou usar o botão Run workflow manualmente uma vez por mês.
9

Incorporar os pesos treinados no jogo

Após alguns ciclos de treinamento, você pode copiar os pesos para o web app:

// No main.js do web app, substitua os valores em AI_WEIGHTS pelos de best-weights.json:
// Abra results/best-weights.json e copie os valores de "weights"

const AI_WEIGHTS = {
  // ... pesos base ...
  completedCityFieldValue: 3.82,  // ← substituído pelo valor treinado
  nearCompleteCityFieldValue: 2.41,
  // etc.
};

Ou usando import dinâmico (se converter o web app para ES modules):

import bestWeights from "../results/best-weights.json" assert { type: "json" };

const AI_WEIGHTS = {
  ...BASE_AI_WEIGHTS,
  ...bestWeights.weights,
};

Dúvidas frequentes

Dúvida Resposta
Preciso pagar alguma coisa? Não. Repositório público tem Actions ilimitado. Repositório privado tem 2.000 min/mês gratuitos.
O workflow falhou no smoke test. O que fazer? Clique no step Smoke test para ver qual dos 17 testes falhou. Há um bug na extração das regras que precisa ser corrigido.
Como parar o treinamento agendado? Na aba Actions, clique no workflow → ⋯ → Disable workflow.
Como ver quanto minuto de Actions já usei? Settings → Billing and plans → Usage this month.
Posso rodar localmente também? Sim. cd "AI Trainer" && npm run smoke valida as regras. npm run evolve treina por 20 minutos localmente.
